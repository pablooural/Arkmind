# Snapshot Estructurado - Proyecto Arkmind

## Arquitectura
- **Estructura modular**: Sistema organizado en módulos independientes con dependencias claras:
  - `snapshot-store`: Almacenamiento de snapshots (dependencias: ninguna).
  - `rollback-engine`: Motor de rollback (dependencias: `snapshot-store`).
  - `op-journal`: Registro de operaciones (dependencias: `rollback-engine`).
  - `runtime-persistence`: Persistencia en runtime (dependencias: `op-journal`).
  - `spec-discrepancies`: Resolución de discrepancias (dependencias: ninguna).
- **Interfaz de usuario**: Componentes clave en `artifacts/ux-arquitecto`:
  - `SnapshotPanel`: Gestión manual de snapshots (crear, expandir, eliminar) con diff visual.
  - `HistoryPanel`: Historial de operaciones en tiempo real con restauración de snapshots.
  - `ConversationPanel`: Interfaz para aceptar/rechazar propuestas IA con transacciones reales.
  - `ChatPanel`: Soporte para streaming SSE de respuestas IA.
- **Backend**: Ruta `POST /api/ai/stream` en `artifacts/api-server` para streaming de tokens.

## Decisiones
- **ADRs aceptados**:
  - `0001`: Implementación de `snapshot-store`.
  - `0002`: Motor de rollback (`rollback-engine`).
  - `0003`: AIProvider opcional (resuelve `Q2`).
  - `0004`: Interfaz de AIProvider y proveedores (`spec-discrepancies`).
  - `0005`: Persistencia en runtime (`runtime-persistence`).
  - `0006`: Registro de operaciones (`op-journal`).
- **Decisiones técnicas**:
  - `MAX_FILE_CONTENT_CHARS=12000`: Límite de caracteres para contenido de archivos.
  - `webFilesystemProvider` directo para evitar ciclos filesystem→transactions.
  - Fallback a `sendAIMessage` si streaming falla.
  - Confirmación inline en SnapshotPanel (no modales).
  - Algoritmo de diff propio sin librerías externas.
  - `AIProvider` opcional: `NoopAIProvider` por defecto, `MistralAIProvider` si hay `apiKey`.

## Problemas / riesgos
- **Abiertos**:
  - `Q1`: ¿Necesidad de versionado de snapshots? Riesgo: Incompatibilidad futura si cambia el formato.
  - `Q3`: Sincronización entre dispositivos. Contexto: Estado en IndexedDB local puede divergir entre dispositivos.
- **Resueltos**:
  - `Q2`: AIProvider opcional (resuelto con ADR 0003).
