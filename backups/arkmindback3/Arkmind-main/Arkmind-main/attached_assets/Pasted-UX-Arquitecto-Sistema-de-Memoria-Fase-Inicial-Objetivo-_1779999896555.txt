UX Arquitecto — Sistema de Memoria (Fase Inicial)

Objetivo

Implementar un sistema de memoria contextual y evolutiva para evitar que cada sesión de IA empiece desde cero.

El objetivo NO es crear una memoria infinita global.

El objetivo es construir:

memoria localizada

memoria contextual

memoria jerárquica

memoria evolutiva

continuidad cognitiva

Filosofía de memoria

La memoria debe vivir:

cerca del contexto

asociada a nodos

resumida

compactable

heredable

La IA no debe depender únicamente del historial del chat.

Cada contexto debe poder:

recordar

resumirse

restaurarse

evolucionar

Arquitectura General

El sistema se dividirá en:

Paso 1 — Working Memory

Paso 2 — Context Memory

Paso 3 — Hierarchical Memory

Paso 4 — Cognitive Snapshots

Paso 5 — Memory Manager

PASO 1 — Working Memory

Objetivo

Mantener el estado cognitivo inmediato de la IA.

Esto representa:

en qué está pensando

qué está haciendo

qué considera importante ahora

Crear:

WorkingMemory 

Ejemplo:

export interface WorkingMemory { focus: string; intent: string; activeResources: string[]; constraints: string[]; keyInsights: string[]; openQuestions: string[]; temporaryNotes: string[]; lastUpdated: number; } 

Uso

Debe actualizarse:

durante conversaciones

durante navegación

durante edición

durante operaciones IA

PASO 2 — Context Memory

Objetivo

Cada carpeta/contexto debe tener memoria propia persistente.

Implementar:

.ai.md 

o:

ContextMemory 

Contenido sugerido

# Context Summary ## Purpose Describe para qué existe este contexto. ## Current Focus En qué se está trabajando. ## Key Decisions - decisión 1 - decisión 2 ## Constraints - constraint 1 - constraint 2 ## Relevant Resources - archivo A - archivo B ## Open Questions - pregunta 1 

Comportamiento

Cuando la IA entra a un contexto:

carga memoria local

resume memoria heredada

fusiona working memory

actualiza contexto cognitivo

PASO 3 — Hierarchical Memory

Objetivo

La memoria debe heredarse por árbol contextual.

Ejemplo

Entrando a:

/core/snapshots 

la IA hereda:

memoria global workspace

memoria /core

memoria /core/snapshots

Resultado

La IA entiende:

dónde está

qué representa ese nodo

qué decisiones ya fueron tomadas

sin releer todo el proyecto.

PASO 4 — Cognitive Snapshots

Objetivo

Guardar snapshots cognitivos antes de cambios importantes.

No solamente snapshots de archivos.

También snapshots del estado mental/contextual.

Crear:

CognitiveSnapshot 

Ejemplo:

export interface CognitiveSnapshot { id: string; contextPath: string; summary: string; workingMemory: WorkingMemory; relatedResources: string[]; createdAt: number; } 

Crear snapshots automáticos antes de:

refactors

escrituras masivas

generación IA

cambios estructurales

merges

ramas

ejecuciones

PASO 5 — Memory Manager

Objetivo

Centralizar:

carga

herencia

compactación

resumen

invalidación

persistencia

Crear:

MemoryManager 

Responsabilidades

Debe poder:

cargar memoria contextual

heredar memoria jerárquica

resumir conversaciones largas

compactar memoria vieja

generar snapshots cognitivos

invalidar memoria obsoleta

sincronizar working memory

Reglas importantes

NO construir todavía:

vector DB compleja

embeddings gigantes

RAG masivo

memoria global infinita

Prioridad real

Primero construir:

memoria contextual viva

memoria jerárquica

snapshots cognitivos

continuidad contextual

Objetivo Final

Lograr que el sistema se comporte como:

un runtime cognitivo persistente 

y no como:

un chat que olvida todo al reiniciar 

Resultado esperado

Cuando el usuario vuelva a un contexto:

la IA recuerda

el contexto recuerda

las decisiones recuerdan

las restricciones recuerdan

el flujo continúa

sin empezar desde cero.
